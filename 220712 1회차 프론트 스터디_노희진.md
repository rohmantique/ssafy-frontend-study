1. 프로그래밍

   1.1 프로그래밍이란?

   ​	컴퓨터에게 실행을 요구하는 일종의 커뮤니케이션. 특정 목적을 달성하기 위해 설계된 알고리즘을 프로그래밍 언어를 사용해 구체적인 프로그램으로 작성하는 과정. 

   1.2 컴파일러와 인터프리터

   - 컴파일러 or 인터프리터가 필요한 이유
     - 인간이 코드를 통해 내린 명령을 수행할 주체는 컴퓨터이다. 사람이 이해할 수 있는 자연어가 아니라 컴퓨터가 이해할 수 있는 언어인 기계어로 명령을 전달해야 한다. 이 과정에서 사람이 이해할 수 있는 약속된 구문으로 구성된 프로그래밍 언어를 사용해 그것을 컴퓨터가 이해할 수 있는 기계어로 변환하는 일종의 번역기가 필요한데, 그 역할을 하는 것이 컴파일러 or 인터프리터다.

   컴파일러(compiler)

   - 전체 파일을 스캔하여 한꺼번에 번역한다.
   - 초기 스캔시간이 오래 걸리지만, 한번 실행 파일이 만들어지고 나면 빠르다.
   - 기계어 번역과정에서 더 많은 메모리를 사용한다.
   - 전체 코드를 스캔하는 과정에서 모든 오류를 한꺼번에 출력해주기 때문에 실행 전에 오류를 알 수 있다.
   - 대표적인 언어로 C, C++, JAVA 등이 있다.

   인터프리터(interpreter)

   - 프로그램 실행시 한 번에 한 문장씩 번역한다.
   - 한번에 한문장씩 번역후 실행 시키기 때문에 실행 시간이 느리다.
   - 컴파일러와 같은 오브젝트 코드 생성과정이 없기 때문에 메모리 효율이 좋다.
   - 프로그램을 실행시키고 나서 오류를 발견하면 바로 실행을 중지 시킨다. 실행 후에 오류를 알 수 있다.
   - 대표적인 언어로 Python, Ruby, Javascript 등이 있다.

3. 변수에 대해서

   3.1 변수란?

   하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름을 뜻한다.

   3.2 식별자란?

   어떤 값을 구별해서 식별할 수 있는 고유한 이름을 말한다.식별자는 값이 아니라 메모리 주소를 기억한다. 쉽게 말해 변수명이라고 기억하면 된다. 

   3.3 변수 선언의 의미는?

   값을 저장하기 위한 메모리 공간을 확보하고 변수 이름과 확보된 메모리 공간의 주소를 연결해서 값을 저장할 수 있게 준비하는 것.
